#######% % markdown comments for various source files
#######% % Dominik Cullmann  
#######%
#######% extract markdown-like comments from (source code) file, convert them 
#######% to valid markdown and run pandoc on it.
#######% Since the comment characters for different languages differ, 
#######% this program can be adjusted to use the comment character used in your 
#######% file by command line arguments.
#######%
#######% copyright: 2014-2017, Dominik Cullmann

#######% maintainer: Dominik cullmann  
#######% email: dominik.cullmann@forst.bwl.de  
#######% 
usage: pydoc.py [-h] [-o NAME_POSTFIX] [-e NAME_PREFIX] [-c COMMENT_CHARACTER]
                [-m MAGIC_CHARACTER] [-v] [-x] [-p] [-n] [-l] [--no-latex]
                file
 
convert markdown-style comments from a file to markdown and html via pandoc.
 
positional arguments:
  file                  The name of the file to convert comments from.
 
optional arguments:
  -h, --help            show this help message and exit
  -o NAME_POSTFIX, --postfix NAME_POSTFIX
                        Change the postfix added to the files created.
                        (default: _contents)
  -e NAME_PREFIX, --prefix NAME_PREFIX
                        Change the prefix added to the files created.
                        (default: )
  -c COMMENT_CHARACTER, --comment COMMENT_CHARACTER
                        Change the comment character. (default: #)
  -m MAGIC_CHARACTER, --magic MAGIC_CHARACTER
                        Change the magic character. (default: %)
  -v, --version         show program's version number and exit
  -x, --example         Give an example and exit.
  -p, --pandoc          Run pandoc on the md file created. (default: False)
  -n, --no-pandoc       Do not run pandoc on the md file created. (default:
                        False)
  -l, --latex           Run LaTex on the tex file created via pandoc.
                        (default: False)
  --no-latex            Run LaTex on the tex file created via pandoc.
                        (default: False)
 
markdown style comments are headed by one or more comment characters giving
the markdown heading level and a magic character marking it as
markdown.
try --example for an example
